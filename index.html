http2的简单分析：

http1/hspdy/http2的简单对比：

  • 1.HTTP/1.1不支持压缩每个请求和相应的HTTP报头。SPDY和后来的HTTP/2旨在使用不同的压缩类型来弥补这个短板。
  • 2.SPDY使用普通的DEFLATE 算法而HTTP/2使用专门为压缩报头而设计的HPACK算法。它使用预定义的token、动态表以 及哈夫曼压缩。

使用http2其性能提高的主要原因就是：

  • 1.使用了链路复用技术，也就是说主要通过创建一个浏览器、服务器之间的永久连接来提高速度，而不是每个信息都 创建一个连接。这将降低了建立连接的时间消耗。另外．
  • 2.其压缩算法好，它使用二进制来传输数据而不是文件，二进制是电脑语言。这说明电脑不需要浪费时间将信息翻译 为它能够理解的形式．

目前nginx原生版本中在1.9.5中已经支持http2了，而且已经将spdy进行移除了． 

server {

listen 80;

location / {

return 301 https://$host$request_uri;　//永久重定向

}

}


server {

listen 443 ssl http2 default_server;

ssl_certificate    server.crt;//相关证书

ssl_certificate_key server.key;//相当于密钥

}

rewrite后面的flag值含义:
301 permanent 代表永久性转移

302 redirect 代表临时性转移

注意其秘钥的管理以及传输的过程之前有说过．

其实301与302在用户看起来是一样的，具体表现是：效果只是一个跳转，浏览器中旧的URL变成了新的URL。页面跳到了这个新 的url指向的地方。对于浏览器而言：302有可能被认为是一种垃圾广告．
